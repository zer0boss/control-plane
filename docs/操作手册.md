# OpenClaw Control Plane 操作手册

## 概述

本手册介绍如何通过 **Control Plane** 集中管理多个 **OpenClaw** 实例（通过 AO 插件 V2 连接）。

### 架构图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Control Plane (管理端)                            │
│                    FastAPI + WebSocket Client                            │
│                         http://localhost:8000                            │
└─────────────────────────────────────────────────────────────────────────┘
                                    │
              ┌─────────────────────┼─────────────────────┐
              ▼ WebSocket           ▼ WebSocket           ▼ WebSocket
┌─────────────────┐       ┌─────────────────┐   ┌─────────────────┐
│   OpenClaw #1   │       │   OpenClaw #2   │   │   OpenClaw #N   │
│  AO Plugin V2   │       │  AO Plugin V2   │   │  AO Plugin V2   │
│  ws://host:port │       │  ws://host:port │   │  ws://host:port │
└─────────────────┘       └─────────────────┘   └─────────────────┘
```

## 一、环境准备

### 1.1 OpenClaw 侧配置

编辑 OpenClaw 配置文件：`C:\Users\Administrator\.openclaw\openclaw.json`

**AO 插件 V2 服务器模式配置：**

```json
{
  "channels": {
    "ao": {
      "enabled": true,
      "connectionMode": "server",
      "listenHost": "0.0.0.0",
      "listenPort": 18080,
      "apiKey": "your-secure-api-key-here",
      "maxConnections": 10,
      "healthCheck": {
        "enabled": true,
        "intervalMs": 30000,
        "timeoutMs": 10000
      },
      "circuitBreaker": {
        "enabled": true,
        "failureThreshold": 5,
        "recoveryTimeoutMs": 30000,
        "halfOpenMaxCalls": 3
      },
      "messageQueue": {
        "enabled": true,
        "maxSize": 1000,
        "persistPath": "./data/ao-queue"
      },
      "metrics": {
        "enabled": true,
        "port": 9090,
        "path": "/metrics"
      },
      "mtls": {
        "enabled": false,
        "autoRotate": true
      }
    }
  }
}
```

**关键配置说明：**

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `connectionMode` | 连接模式，必须为 `server` | `"server"` |
| `listenHost` | 监听地址 | `"0.0.0.0"` |
| `listenPort` | WebSocket 监听端口 | `18080` |
| `apiKey` | 认证密钥（重要！） | `"your-secure-api-key"` |
| `maxConnections` | 最大连接数 | `10` |

### 1.2 启动 OpenClaw

确保 OpenClaw 已启动，AO 插件会自动开始监听 WebSocket 连接。

验证监听状态：
```bash
# Windows
netstat -an | findstr 18080

# Linux/Mac
netstat -tlnp | grep 18080
```

### 1.3 启动 Control Plane

```bash
# 进入后端目录
cd control-plane/backend

# 安装依赖
pip install -r requirements.txt

# 启动服务
python -m app.main
```

服务启动后访问：
- API 文档：http://localhost:8000/docs
- 健康检查：http://localhost:8000/health

## 六、完整操作流程示例

### 场景：通过 Control Plane 向 OpenClaw 发送消息

```bash
#!/bin/bash

# 步骤 1: 注册 OpenClaw 实例
INSTANCE=$(curl -s -X POST http://localhost:8000/api/v1/instances \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试实例",
    "host": "127.0.0.1",
    "port": 18080,
    "channel_id": "ao",
    "credentials": {
      "auth_type": "token",
      "token": "openclaw-ao-v2-server-key"
    }
  }')

INSTANCE_ID=$(echo $INSTANCE | jq -r '.id')
echo "实例ID: $INSTANCE_ID"

# 步骤 2: 建立连接
curl -s -X POST "http://localhost:8000/api/v1/instances/$INSTANCE_ID/connect"
echo "连接已建立"

# 步骤 3: 创建会话
SESSION=$(curl -s -X POST http://localhost:8000/api/v1/sessions \
  -H "Content-Type: application/json" \
  -d "{
    \"instance_id\": \"$INSTANCE_ID\",
    \"target\": \"test-session\",
    \"context\": {}
  }")

SESSION_ID=$(echo $SESSION | jq -r '.id')
echo "会话ID: $SESSION_ID"

# 步骤 4: 发送消息
MESSAGE=$(curl -s -X POST "http://localhost:8000/api/v1/sessions/$SESSION_ID/messages" \
  -H "Content-Type: application/json" \
  -d '{
    "content": "你好，OpenClaw！",
    "stream": false
  }')

echo "消息已发送: $(echo $MESSAGE | jq -r '.id')"

# 步骤 5: 查看消息历史
curl -s "http://localhost:8000/api/v1/sessions/$SESSION_ID/messages" | jq
```

## 八、故障排查

### 8.1 连接失败

**现象：** `POST /connect` 返回 `success: false`

**排查步骤：**

1. 检查 OpenClaw 是否已启动
   ```bash
   curl http://localhost:18080/health
   ```

2. 检查网络连通性
   ```bash
   telnet <host> <port>
   ```

3. 验证 API Key 是否正确

4. 检查防火墙设置

### 8.2 认证失败

**现象：** 返回 401 Unauthorized

**解决方案：**
- 确认 OpenClaw 配置中的 `apiKey` 与 Control Plane 实例配置中的 `token` 一致
- 检查 token 是否包含特殊字符需要转义

### 8.3 消息发送超时

**现象：** 发送消息返回超时错误

**解决方案：**
- 检查 OpenClaw 实例状态：`GET /api/v1/instances/{id}/health`
- 增加超时时间：`timeout: 120`
- 检查网络延迟

### 8.4 查看日志

**Control Plane 日志：**
```bash
# 查看实时日志
tail -f control-plane/logs/app.log
```

**OpenClaw AO 插件日志：**
```bash
# 在 OpenClaw 日志中过滤 AO 相关日志
tail -f ~/.openclaw/logs/openclaw.log | grep "AO"
```

## 九、安全配置建议

### 9.1 API Key 安全

1. **生产环境必须使用强密码**
   ```json
   "apiKey": "使用随机生成的32位以上字符串"
   ```

2. **定期更换 API Key**
   - 建议每 90 天更换一次
   - 更换后需重新注册实例

3. **使用环境变量**
   ```bash
   # 不要硬编码在配置文件中
   export AO_API_KEY="your-secure-key"
   ```

### 9.2 启用 mTLS（可选）

对于更高安全要求的环境：

```json
{
  "channels": {
    "ao": {
      "mtls": {
        "enabled": true,
        "certPath": "/path/to/cert.pem",
        "keyPath": "/path/to/key.pem",
        "caPath": "/path/to/ca.pem",
        "autoRotate": true
      }
    }
  }
}
```
